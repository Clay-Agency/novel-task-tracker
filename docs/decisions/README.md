# Decision log (DR / ADR)

This folder is our lightweight **decision log** (ADR-style): short records of important project decisions so we can understand *what we decided* and *why* later.

## When to add a decision record

Add a DR when:
- we close an issue labeled **`needs-decision`**, or
- a choice materially affects scope, UX, architecture, or process.

## Naming convention

- File name: `DR-###-short-kebab-title.md` (example: `DR-001-tefq-pilot.md`)
- Keep one decision per file.

## Status values

Use one of:
- `Proposed`
- `Accepted`
- `Rejected`
- `Superseded` (and link to the newer DR)
